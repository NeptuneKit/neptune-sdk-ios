# 2026-03-27

- iOS 导出服务已从占位 `RootView` 切换为真实 UIKit 视图树采集，默认通过 `UIApplication.shared.connectedScenes` + `UIWindow.windows` 遍历视图层级，不再依赖手动锚点。
- 客户端已收口为 inspector-only：`/v2/ui-tree/snapshot` 不再对外暴露，网关 raw 上报固定使用 `POST /v2/ui-tree/inspector`。
- 新增 `/v2/ui-tree/inspector`，返回 `InspectorSnapshot`，其中 `payload` 使用 JSON object/array 编码，不再字符串化 JSON。
- 新增可注入的 `NeptuneViewTreeCollecting`，便于测试替换采集器并保持 HTTP 路由可测。
- 字体相关字段已统一按 iOS 逻辑像素语义输出，`style` 新增 `typographyUnit = "dp"`，并补充 `fontSize`、`lineHeight`、`letterSpacing`；`inspector` 的 `style` attrs 同步透出同一组字段。
- 后续补齐了源语义字段：`sourceTypographyUnit = "pt"`、`platformFontScale`、`fontWeightRaw`；前端既能继续用统一 `dp` 渲染，也能用原始语义做更精细的回放和对齐。
- 新增 UIKit 侧的 collector 采集断言，直接验证 `UILabel` 的字体、行高、字距和原始权重元数据会被抓到。
- 验证：`cd neptune-sdk-ios && swift test --filter ExportHTTPServerTests` 通过（9 tests）；`cd neptune-sdk-ios && swift test --filter ViewTreeTypographyTests` 通过（2 tests）；`cd neptune-sdk-ios && swift test` 在当前 macOS SwiftPM 环境里所有可见 suites 已通过，但进程最后返回了 `signal code 5`，更像是 runner 环境噪音，非本次新增断言失败。
- 示例工程新增一个可从首页进入的复杂二级页 `DeepDiveViewController`，页面包含胶囊标签、指标矩阵、时间轴列表和诊断卡片，便于和 Harmony DeepDive 对齐并让视图树采集链路采到更真实的 UIKit 层级。
