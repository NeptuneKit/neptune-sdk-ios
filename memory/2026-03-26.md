# 2026-03-26

- 结论：`Release XCFramework` 失败根因是 `scripts/build-xcframework.sh` 的运行时依赖检查将 `otool -L` 输出中的静态成员条目（`binary(member.o)`）误判为动态依赖，触发 `exit 1`。
- 修复：在依赖检查中显式忽略 `"$binary_path(... .o)"` 形式条目，并新增 `--check-runtime-dependencies-only` 以便单测可直接验证依赖检查逻辑。
- 回归：新增两条脚本测试，分别验证“静态成员应被忽略（通过）”和“三方动态库应失败（拦截）”。
- 追加修复：GitHub Runner 中 `otool -L` 条目实际为 `binary(member.o):`（包含尾部冒号），已在过滤条件中同时支持 `)` 与 `):` 两种格式。
