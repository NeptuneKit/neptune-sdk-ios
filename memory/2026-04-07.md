## iOS 最低部署版本调整

- 决策：`neptune-sdk-ios` 源码包、`neptune-sdk-xcframework` 二进制包，以及 `Examples/simulator-app` 的 deployment target 统一下调为 `iOS 16+`。
- 原因：业务侧需要在 iOS 16 环境通过 SPM / XCFramework 接入，避免源码包与二进制包最低版本不一致。

## XCFramework 发布链自动同步

- 决策：`neptune-sdk-ios` 的 `Release XCFramework` workflow 在生成 release 资产后，继续自动同步 `neptune-sdk-xcframework` 仓库的 Release、`Package.swift` 与 README。
- 实现：新增 `scripts/sync-xcframework-wrapper.sh`，由 workflow 读取 zip 的 `.sha256` 并回写 wrapper 仓库元数据，再自动提交推送。
- 风险门禁：跨仓库同步依赖 `NEPTUNE_XCFRAMEWORK_REPO_TOKEN`；若 secret 缺失，workflow 会显式失败，避免二进制分发仓库停留在旧版本。
