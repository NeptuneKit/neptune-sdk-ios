# 2026-03-25

## Simulator App 自动注册闭环

- `Examples/simulator-app` 现在会在 `viewDidLoad` / `viewDidAppear` 的首次进入时自动启动 gateway registration client，不再依赖手工点 `Discover Gateway` 或环境变量开关。
- registration client 通过日志代理输出 `gateway registration started`、`gateway registration discovery success/failure`、`gateway registration success/failure`，失败后按 `5s` 续约/重试间隔继续尝试。
- 新增 `SimulatorAppTests` 覆盖自动注册入口触发和关键日志格式，避免后续回归成“只写按钮不启动链路”。

## XCFramework 打包脚本修正

- 修复 `scripts/build-xcframework.sh` 在 `set -u` 下空数组展开导致的异常退出：`ALLOW_RUNTIME_DEPENDENCIES` 为空时不再触发 `unbound variable`。
- 修复运行时依赖检查误判：静态库成员（如 `NeptuneSDKiOS(SomeObject.o):`）不再被当作动态依赖。
- 验证结果：
  - `NeptuneSDKiOS.xcframework` 可生成于 `.build/artifacts/NeptuneSDKiOS.xcframework`。
  - 基于当前产物执行依赖检查逻辑，结果为 `NO_LEAKS`。

## Release 自动挂载 XCFramework

- 新增 `scripts/build-release-assets.sh`，用于 release 场景统一产出：
  - `NeptuneSDKiOS-<tag>.xcframework.zip`
  - `NeptuneSDKiOS-<tag>.xcframework.zip.sha256`
- 新增 workflow `.github/workflows/release-xcframework.yml`：
  - 触发 `release published` 或 `workflow_dispatch`
  - 自动构建并挂载发布附件到对应 GitHub Release。
- 通过 `XCFrameworkScriptTests` 为发布脚本补充 TDD 覆盖：`--help`、非法 tag、`--dry-run` 成功路径。
- 版本号策略调整：`build-release-assets.sh` 的 `--tag` 改为可选，不传时默认当天日期（如 `2026.3.25`）；支持 `vX.Y.Z` 与 `YYYY.M.D` 两种格式。
- workflow 调整：`workflow_dispatch` 未传 `tag_name` 时，使用当天日期作为 release 版本，但 checkout 当前提交（`GITHUB_SHA`）避免因不存在日期 tag 导致 checkout 失败。
- 同日多版本策略：若 `YYYY.M.D` 已存在，则默认版本自动递增为 `YYYY.M.D.1`、`YYYY.M.D.2`；脚本按“本地 tag + 已有发布 zip 资产”检测冲突，workflow 按“远端 tags”检测冲突。
